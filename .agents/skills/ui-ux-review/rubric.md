# UI/UX Review Rubric

Score each category from 1 to 10.

## 1. Task Support

Does the UI help the user complete the primary task?

10:
- Primary task is obvious
- Main action is easy to find
- User can complete the flow with minimal confusion

5:
- Primary task is possible but unclear
- Some actions compete with each other

1:
- UI does not support the stated task

## 2. Information Hierarchy

Does the UI make the most important information most visible?

10:
- Clear primary, secondary, and tertiary information
- Key values are prominent and labeled

5:
- Important information exists but lacks hierarchy

1:
- User cannot tell what matters most

## 3. Clarity

Are labels, data, states, and actions unambiguous?

10:
- All key data has clear meaning
- Dates, amounts, and statuses are understandable

5:
- Some labels are vague

1:
- Important data can be misinterpreted

## 4. Mobile Usability

Does the mobile UI feel intentionally designed?

10:
- Layout fits mobile behavior
- Tables become cards
- Filters move into sheets or drawers
- Touch targets are comfortable

5:
- Mostly usable but some desktop patterns leak into mobile

1:
- Desktop UI is simply squeezed into mobile

## 5. Navigation

Can the user understand where they are and where to go next?

10:
- Current section is obvious
- Navigation is predictable
- Scope switchers are clear

5:
- Navigation is present but labels or hierarchy are unclear

1:
- User may get lost

## 6. Visual Consistency

Are spacing, typography, icons, cards, and states consistent?

10:
- Consistent spacing, type scale, and component usage

5:
- Minor inconsistencies

1:
- UI feels assembled from unrelated pieces

## 7. Accessibility

Can users with different abilities use the interface?

10:
- Good contrast
- Clear labels
- Touch targets are large enough
- Important meaning is not color-only

5:
- Some accessibility concerns

1:
- Major accessibility risks

## 8. Responsive Readiness

Will the layout adapt cleanly across breakpoints?

10:
- Explicit behavior for mobile, tablet, and desktop

5:
- Works on one breakpoint but uncertain elsewhere

1:
- No clear responsive strategy

## 9. State Handling

Are empty, loading, error, success, and partial-data states handled?

10:
- States are meaningful and context-aware

5:
- Some states are missing or generic

1:
- States are misleading or absent

## 10. Implementation Risk

Can the suggested UI be implemented consistently with project constraints?

10:
- Fits existing component system and patterns

5:
- Requires some custom work

1:
- Requires large unplanned redesign or new system
